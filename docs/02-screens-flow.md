# FIT 헤어컨설팅 - 화면 플로우

## 전체 뷰 구조

앱은 3가지 뷰(View)로 구성됩니다.

```
App
├── consultation (기본 뷰) — 컨설팅 12단계 스텝
├── client-list            — 고객 목록
└── client-detail          — 고객 상세 + 방문 이력
```

---

## 컨설팅 플로우 (12단계)

### Step 0: 인트로 (IntroStep)

- **목적**: 앱 진입 화면
- **UI**: 브랜드명 "FIT" 대형 타이포, 설명 텍스트, 2개 버튼
- **액션**:
  - `Start` 버튼 → Step 1로 이동
  - `고객 목록` 버튼 → client-list 뷰로 전환
- **특이사항**: Progress Bar 없음

---

### Step 1: 고객 기본 정보 (ClientInfoStep)

- **목적**: 컨설팅 대상 고객 정보 입력
- **입력 필드**:

| 필드 | 타입 | 옵션 | 필수 |
|------|------|------|------|
| 고객명 | text input | - | ✅ |
| 연락처 | tel input | 010-0000-0000 형식 | ✅ |
| 연령대 | radio | 20-30대, 40대, 50대, 60대 이상 | ✅ |
| 성별 | radio | 여자(female), 남자(male) | ✅ |

- **유효성**: 4개 필드 모두 입력 시 다음 버튼 활성화

---

### Step 2: TODAY KEYWORD (TodayKeywordStep)

- **목적**: 오늘의 상태와 선호 이미지 파악 (성별에 따라 옵션 다름)
- **입력 필드**:

| 필드 | 타입 | 옵션 |
|------|------|------|
| 얼굴 보완 희망 부위 | checkbox (복수) | 이마, 광대, 볼 옆 라인, 턱선, 얼굴 전체 비율, 얼굴이 길어 보이는 느낌, 얼굴이 커 보이는 느낌, 기타 |
| 얼굴 보완 기타 메모 | textarea | "기타" 선택 시 표시 |
| 헤어 고민 | checkbox (복수) | 앞머리, 정수리 볼륨, 옆 볼륨, 스타일 변화, 모발 손상, 모질, 두피, 기타 |
| 헤어 고민 기타 메모 | textarea | "기타" 선택 시 표시 |
| 선호 이미지 키워드 | checkbox (복수) | 여성: 귀여운/사랑스러운, 어려보이는, 프레시한, 청초한, 부드러운/여성스러운, 단아한, 시크/세련된, 우아한/클래식한, 자연스러운, 지적인/현대적인 |
| | | 남성: 귀여운/어려보이는, 단정한, 부드러운/젠틀한, 강한/카리스마있는, 시크/세련된, 클래식한, 자연스러운, 지적인/현대적인 |

---

### Step 3: 패션 스타일 (FashionStyleStep)

- **목적**: 평소 패션 스타일 파악
- **입력**: 스타일 카드 선택 (복수 선택, 성별 맞춤)
- **표시**: 이미지 카드 형태

---

### Step 4: 로딩 (LoadingStep)

- **목적**: AI 분석 중 시각적 피드백 (실제 AI 연산 없음, UI 연출용)
- **표시**: 고객명 + 로딩 애니메이션
- **자동 전환**: 일정 시간 후 다음 스텝으로 자동 이동

---

### Step 5: 디자인 요약 (DesignSummaryStep)

- **목적**: 앞서 입력한 키워드/스타일 기반으로 중간 요약 표시
- **표시**: 입력한 컨설팅 데이터 요약 (읽기 전용)

---

### Step 6: 얼굴 & 이미지 타입 분석 (FaceImageTypeStep)

- **목적**: 디자이너가 고객 얼굴 이목구비를 분석하여 이미지 타입 결정
- **입력 필드**:

| 필드 | 옵션 (WARM → NEUTRAL → COOL) |
|------|------|
| 윤곽 | 둥근 / 중간 / 각진 |
| 눈썹 | 둥근 내려간 / 중간 / 각진 올라간 |
| 눈 | 둥근 내려간 눈 / 중간 / 직선 올라간 눈 |
| 코 | 둥근 코 / 중간 / 뾰족한 코 |
| 입술 | 도톰 / 중간 / 얇은 |
| 전체 이미지 타입 | WARM / NEUTRAL / COOL |

- **이미지 타입 해석**:
  - WARM: 귀여운·사랑스러운·부드러운 (여성) / 귀여운·부드러운·친근한 (남성)
  - NEUTRAL: 자연스러운·균형잡힌
  - COOL: 세련된·시크한·우아한 (여성) / 세련된·시크한·지적인 (남성)
- **시각 자료**: 성별별 WARM/COOL 예시 이미지 표시

---

### Step 7: 모발 상태 (HairConditionStep)

- **목적**: 현재 모발 컨디션 진단
- **입력 필드**:

| 필드 | 타입 | 옵션 |
|------|------|------|
| 손상도 | radio | 건강, 약손상, 중손상, 강손상, 극손상, 초극손상 |
| 모질 | checkbox (복수) | (모질 타입들) |
| 두께 | radio | 가는편, 보통, 굵은편 |
| 밀도 | radio | 적음, 보통, 많음 |
| 웨이브 | radio | 직모, 약웨이브, 강웨이브 |

- **고손상 시 처리**: todayDesign 단계에서 "손상도 회복 우선" 안내 표시

---

### Step 8: 헤어스타일 제안 (HairStyleProposalStep)

- **목적**: 오늘의 헤어스타일 길이 방향 제안
- **입력**:
  - 제안 길이 선택
  - 레퍼런스 이미지 (선택)

---

### Step 9: TODAY DESIGN 정리 (TodayDesignStep)

- **목적**: 오늘 진행한 디자인 결과 기록 + 다음 방문 방향 설정
- **TODAY DESIGN 입력**:

| 항목 | 타입 | 옵션 |
|------|------|------|
| 길이 | checkbox (복수) | 숏, 단발, 미디움, 롱 |
| 앞머리 | checkbox (복수) | 없음, 시스루, 풀뱅, 사이드뱅, 스틱뱅, 처피뱅, 남자 |
| 컬/질감 | checkbox (복수) | 스트레이트, C컬, S컬, CS컬, 웨이브 |
| 컬러 | checkbox (복수) | 톤다운, 톤업, 뿌리, 탈색, 현재 상태 유지 |
| 각 항목별 메모 | textarea | 세부사항 기록 |

- **NEXT DIRECTION 입력**:

| 항목 | 타입 | 서브 옵션 |
|------|------|------|
| 길이 변화 | checkbox + 확장 | 롱, 미디움, 단발, 숏 |
| 컬러 변화 | checkbox + 확장 | 톤업, 톤다운, 탈색, 특수컬러 |
| 앞머리 디자인 변경 | checkbox | - |
| 펌 / 볼륨 추가 | checkbox | - |
| 손상도 회복 후 디자인 확장 | checkbox | - |
| 이미지 변화 컨설팅 | checkbox | - |

- **컨텍스트 안내**: 이미지 타입 + 고객명 + 고손상 여부에 따른 맞춤 메시지

---

### Step 10: AFTER NOTE (AfterNoteStep)

- **목적**: 컨설팅 최종 정리 + PDF 생성
- **입력**:

| 필드 | 설명 |
|------|------|
| 디자이너명 | 담당 디자이너 이름 입력 (필수) |
| AFTER DESIGN MEMO | 오늘 핵심 포인트, 고객 반응, 다음 방문 참고사항 (필수) |
| 디자인 사이클 가이드 | 향후 방문 스케줄 설정 (월별 서비스 계획) |

- **표시 (읽기 전용)**: 고객명, 방문일, 이미지 타입, 손상도, TODAY DESIGN 요약, NEXT DIRECTION
- **출력 기능**:
  - `완료` 버튼 → 컨설팅 저장 후 Step 11 이동
  - (숨겨진) PDF 다운로드 / 인쇄 기능

---

### Step 11: 완료 (CompletionStep)

- **목적**: 컨설팅 완료 확인 + 재시작
- **표시**: 완료 메시지
- **액션**:
  - `처음으로` → 모든 데이터 초기화, Step 0으로 이동

---

## 고객 관리 뷰

### client-list 뷰 (ClientListStep)

- **진입**: IntroStep에서 "고객 목록" 버튼 클릭
- **기능**:
  - 전체 고객 목록 조회 (Supabase customers 테이블)
  - 이름/전화번호 검색
  - 고객 카드 클릭 → client-detail 뷰로 이동
- **뒤로**: consultation 뷰(Intro)로 복귀

### client-detail 뷰 (ClientDetailStep)

- **진입**: client-list에서 고객 선택
- **기능**:
  - 선택 고객 기본 정보 표시
  - 해당 고객의 방문 이력 (consultations) 조회
  - `새 컨설팅 시작` → 해당 고객 정보를 미리 채운 채로 Step 1로 이동
- **뒤로**: client-list 뷰로 복귀

---

## Progress Bar 규칙

- Step 1 ~ Step 10 구간에서만 표시
- Step 0 (Intro), Step 11 (Completion)에서는 숨김
- `totalSteps = 12` 기준으로 진행률 계산

---

## 화면 전환 요약

```
[Intro]
  ├── Start → [Step 1: 고객정보]
  │             → [Step 2: TODAY KEYWORD]
  │             → [Step 3: 패션스타일]
  │             → [Step 4: 로딩]
  │             → [Step 5: 디자인요약]
  │             → [Step 6: 얼굴타입분석]
  │             → [Step 7: 모발상태]
  │             → [Step 8: 헤어스타일제안]
  │             → [Step 9: TODAY DESIGN]
  │             → [Step 10: AFTER NOTE]
  │             → [Step 11: 완료] → [Intro] (리셋)
  │
  └── 고객 목록 → [client-list]
                    └── 고객 선택 → [client-detail]
                                      └── 새 컨설팅 시작 → [Step 1] (고객정보 자동입력)
```
